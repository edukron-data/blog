Interview Questions & Answers

Q1: Star schema vs snowflake — interview trade-offs.

Answer: Star: denormalized dimensions, simpler joins, faster BI queries. Snowflake: normalized dimensions, less redundancy, more joins. Lakehouse often uses star/silver-gold with dimension tables in Delta.

Q2: How would you implement SCD Type 2 in SQL?

Answer: Track effective_start, effective_end, is_current flag or end_date sentinel. On change, close current row and insert new version. Delta MERGE is common on Azure; discuss surrogate keys and indexing current rows.

Q3: What is parameter sniffing and how do you mitigate it?

Answer: Optimizer caches plan for first parameter values which may be suboptimal for others. Mitigations: OPTIMIZE FOR UNKNOWN, recompile hints, Query Store plan forcing, local variable workaround (careful), updated statistics.

Q4: Explain the difference between WHERE and HAVING.

Answer: WHERE filters rows before aggregation; HAVING filters groups after GROUP BY. You cannot reference aggregate aliases in WHERE without a subquery/CTE. Example: HAVING SUM(amount) > 1000 for high-spend customers.

Q5: When would you use a window function instead of GROUP BY?

Answer: When you need row-level detail plus aggregates — running totals, rankings, lag/lead comparisons. GROUP BY collapses rows; windows preserve grain. Example: revenue share per order within customer without losing order rows.

Q6: How do indexes affect INSERT/UPDATE performance?

Answer: Indexes speed reads but add write overhead maintaining B-tree structures. Too many indexes hurt write-heavy OLTP. Covering indexes reduce key lookups. Mention included columns and selective filters.

Q7: Describe isolation levels and a phantom read scenario.

Answer: READ UNCOMMITTED allows dirty reads; READ COMMITTED prevents them; REPEATABLE READ prevents non-repeatable reads; SERIALIZABLE prevents phantoms. Explain trade-off between consistency and concurrency; SQL Server/Snapshot isolation variants.

Q8: Write an approach for top-N per group without correlated subqueries.

Answer: Use ROW_NUMBER() OVER (PARTITION BY group ORDER BY metric DESC) in CTE, filter rn <= N. Discuss index on (group, metric) for performance vs RANK ties handling.

Q9: What follow-ups might interviewers ask after transactions & acid questions?

Answer: Expect depth on edge cases, trade-offs between joins, window functions, query plans, indexing, failure recovery, and how your answer changes at scale or under compliance constraints (HIPAA, SOC2).

Q10: How do you communicate transactions & acid answers clearly in a live interview?

Answer: Use structure: definition → example → trade-offs → production consideration. Write clean code or SQL incrementally, state assumptions, and narrate your thinking while validating with a small test case.
