Interview Questions & Answers

Q1: Transformer attention in one minute.

Answer: Attention computes weighted sum of values where weights = softmax(QK^T/√d). Each token attends to all tokens capturing context. Self-attention in encoder; masked in decoder for autoregressive generation.

Q2: RAG vs fine-tuning decision matrix.

Answer: RAG for dynamic knowledge, citation needs, lower training cost. Fine-tune for style/format/domain terminology, task-specific behavior. Often combine: fine-tuned model + RAG for facts.

Q3: MLOps: what do you monitor post-deployment?

Answer: Data drift, prediction drift, latency, error rates, business KPIs, ground truth delay pipelines, alert on feature distribution shift, automated retrain triggers, model version rollback.

Q4: Explain vanishing gradients in RNNs.

Answer: Repeated multiplication of gradients <1 across time steps shrinks signal. LSTM/GRU gates mitigate. Transformers replace recurrence with attention for long-range deps.

Q5: Design churn prediction system end-to-end.

Answer: Feature store from billing/product usage, label definition (churn in 30d), time-based train/test split preventing leakage, class imbalance handling, SHAP for explainability, batch scoring pipeline, CRM integration, monitor performance monthly.

Q6: LLM evaluation beyond BLEU.

Answer: Human rubrics, LLM-as-judge with calibration, task-specific accuracy (tool call success), toxicity/faithfulness checks, RAG citation recall, regression suites on golden prompts, online A/B testing.

Q7: Responsible AI — fairness intervention example.

Answer: Audit metrics across demographic groups, reweight training data, threshold adjustment per group cautiously (legal review), document model cards, human review for high-stakes decisions, logging for appeals.

Q8: Bias-variance tradeoff — explain with example.

Answer: High bias: underfit, model too simple. High variance: overfit, memorizes noise. Use validation curves, regularization, more data, simpler models, or ensemble methods. Connect to cross-validation metrics trends.

Q9: What follow-ups might interviewers ask after ml fundamentals questions?

Answer: Expect depth on edge cases, trade-offs between scikit-learn, deep learning, NLP, LLMs, MLOps, failure recovery, and how your answer changes at scale or under compliance constraints (HIPAA, SOC2).

Q10: How do you communicate ml fundamentals answers clearly in a live interview?

Answer: Use structure: definition → example → trade-offs → production consideration. Write clean code or SQL incrementally, state assumptions, and narrate your thinking while validating with a small test case.
