Interview Questions & Answers

Q1: Explain list vs tuple vs set — when would you choose each?

Answer: Lists are ordered mutable sequences for homogeneous collections you modify. Tuples are immutable ordered records — use for fixed-size data, dict keys, and function returns. Sets are unordered unique collections for membership tests and deduplication. Discuss time complexity: O(1) average set lookup vs O(n) list scan.

Q2: What is the GIL and how does it affect concurrency?

Answer: The Global Interpreter Lock allows only one thread to execute Python bytecode at a time in CPython. CPU-bound parallelism needs multiprocessing or native extensions; I/O-bound work benefits from threading or asyncio because threads release the GIL during I/O waits.

Q3: How do decorators work under the hood?

Answer: Decorators are higher-order functions applying wrapper(callable). @decorator desugars to func = decorator(func). Mention functools.wraps to preserve metadata, parameterized decorators (decorator factory), and common uses: timing, auth, caching, retry.

Q4: Describe Python memory management.

Answer: CPython uses reference counting plus cyclic garbage collector for containers. Discuss __slots__ for memory savings, weak references, and that large objects may use pymalloc arenas. For interviews, connect to profiling with tracemalloc.

Q5: How would you design a rate-limited API client in Python?

Answer: Use asyncio + semaphore or token bucket algorithm, exponential backoff on 429, structured logging, configurable limits per endpoint, and unit tests with mocked clock. Mention tenacity for retries and httpx/aiohttp for async I/O.

Q6: What are common pitfalls with mutable default arguments?

Answer: def append_item(x, bucket=[]) shares one list across calls. Fix with None sentinel: bucket = bucket or []. This is a frequent bug in interview coding — always explain the fix.

Q7: Explain *args and **kwargs with a practical example.

Answer: *args collects positional extras as tuple; **kwargs collects keyword extras as dict. Use in wrapper functions forwarding to inner APIs. Example: logging decorator or Django view wrapper.

Q8: How do you test async code with pytest?

Answer: Use pytest-asyncio, mark tests @pytest.mark.asyncio, mock aiohttp responses with aioresponses, and test cancellation/timeouts. For integration tests, spin ephemeral services or use testcontainers.

Q9: System design: Scenario: System Design — outline your architecture.

Answer: Structure answer: requirements clarifying questions, high-level diagram (ingress, compute, storage, observability), data flow, scaling strategy, failure modes, security, cost estimate, and phased MVP rollout.

Q10: What follow-ups might interviewers ask after scenario: system design questions?

Answer: Expect depth on edge cases, trade-offs between Python 3, data structures, concurrency, pytest, failure recovery, and how your answer changes at scale or under compliance constraints (HIPAA, SOC2).
