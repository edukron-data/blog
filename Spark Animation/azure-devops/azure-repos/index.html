Path Phases

Phase 1: Git & Setup

Git model, repo setup, branching, commits.

Phase 2: Collaboration

PRs, reviews, policies, merges, conflicts, tags, hooks.

Phase 3: Enterprise Repos

Permissions, forks, IDE integration, capstone.

Prerequisites

  • Command-line familiarity
  • Azure DevOps organization access

What You Will Achieve

  • ✓ Implement trunk-based or GitFlow with Azure Repos
  • ✓ Configure branch policies and PR templates
  • ✓ Resolve merge conflicts and automate with hooks

All Topics

    Commits, branches, remotes, and the Git object model.

  2. Azure Repos Setup

    Creating repos, cloning, and authentication methods.

  3. Branching Strategy

    GitFlow, trunk-based, and feature branch workflows.

  4. Commit Best Practices

    Atomic commits, messages, and conventional commits.

  5. Pull Requests

    Creating PRs, descriptions, and linking work items.

  6. Code Review

    Review checklist, constructive feedback, and approval policies.

  7. Branch Policies

    Required reviewers, build validation, and merge types.

  8. Merge Strategies

    Merge commit, squash, rebase, and when to use each.

  9. Conflict Resolution

    Merge conflicts, rebase conflicts, and resolution tools.

  10. Tags & Releases

    Semantic versioning, Git tags, and release notes.

  11. Hooks & Webhooks

    Server hooks, service hooks, and automation triggers.

  12. Repo Permissions

    Contributors, readers, and security group integration.

  13. Fork Workflows

    Forking, syncing, and contributing to upstream repos.

  14. IDE Integration

    VS Code, Visual Studio, and Git credential management.

  15. Repos Capstone

    Set up a repo with branch policies and PR workflow.
