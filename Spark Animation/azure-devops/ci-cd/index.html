Path Phases

Phase 1: CI/CD Foundations

Definitions, CI/CD principles, pipeline design, branching, automated testing.

Phase 2: Delivery Patterns

Artifact promotion, environments, strategies, feature flags, GitOps, DORA metrics.

Phase 3: Organizational CI/CD

Security in pipeline, compliance, monorepo patterns, maturity models, capstone.

Prerequisites

  • Azure Pipelines or similar CI tool exposure helpful

What You Will Achieve

  • ✓ Design CI/CD workflows for microservices
  • ✓ Choose deployment strategies for risk tolerance
  • ✓ Measure DORA metrics and improve release frequency

All Topics

  1. CI/CD Overview

    Continuous integration, delivery, and deployment definitions.

  2. CI Principles

    Frequent commits, automated builds, and fast feedback loops.

  3. CD Principles

    Automated releases, deployment pipelines, and rollback.

  4. Pipeline Design

    Stage ordering, artifact flow, and environment promotion.

  5. Branching for CI/CD

    Feature branches, main protection, and release branches.

  6. Automated Testing in CI

    Unit, integration, and E2E tests in the pipeline.

  7. Deployment Strategies

    Blue-green, canary, rolling, and feature flags.

  8. Infrastructure CI/CD

    Terraform/Bicep in pipelines and IaC validation.

  9. Database CI/CD

    Schema migrations, flyway, and database deployment patterns.

  10. Container CI/CD

    Build, scan, push, and deploy container images.

  11. GitOps

    Declarative deployments, Argo CD, and Flux concepts.

  12. Feature Flags

    Progressive delivery and LaunchDarkly-style rollouts.

  13. Rollback Strategies

    Automated rollback, health checks, and smoke tests.

  14. Environment Configuration

    Dev, staging, prod parity and config management.

  15. Secrets in CI/CD

    Key Vault integration, secret rotation, and pipeline security.

  16. Compliance Gates

    Audit trails, approval workflows, and change management.

  17. DORA Metrics

    Deployment frequency, lead time, MTTR, and change failure rate.

  18. CI/CD Maturity Model

    Assessing and improving your delivery pipeline maturity.

  19. Trunk-Based Development

    Short-lived branches and continuous integration at scale.

  20. CI/CD Capstone

    Design a CI/CD strategy for a microservices application.
